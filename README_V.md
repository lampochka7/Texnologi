# PostgreSQL Практика: Пагинация, Сортировка и Фильтрация данных

[![PostgreSQL](https://img.shields.io/badge/PostgreSQL-16-blue.svg)](https://www.postgresql.org/)


## 📋 Описание проекта

Данный проект представляет собой коллекцию практических заданий по работе с реляционными базами данных в PostgreSQL. Основное внимание уделено трем ключевым аспектам работы с данными:

- **Сортировка** — упорядочивание результатов запросов по одному или нескольким полям
- **Фильтрация** — отбор данных по заданным условиям
- **Пагинация** — разбиение результатов на страницы для удобного отображения

Проект включает 6 заданий различной сложности, от простых выборок до комбинированных запросов с многоуровневой сортировкой и пагинацией. Каждое задание решает конкретную практическую задачу из области электронной коммерции и управления заказами.

## 📊 Содержание заданий

### Задание 1. Топ-2 самых дорогих товара
Выбор двух товаров с наибольшей ценой из каталога. Запрос демонстрирует базовую сортировку по убыванию и ограничение количества результатов.

### Задание 2. "Вторая страница" товаров
Реализация пагинации каталога товаров с размером страницы 2 элемента. Запрос показывает вторую страницу результатов (товары 3 и 4) с использованием стабильной сортировки для предсказуемого порядка.

### Задание 3. Топ-3 заказов по сумме
Определение трех заказов с максимальной суммой. Запрос использует многоуровневую сортировку: сначала по сумме заказа, затем по идентификатору для обеспечения стабильности.

### Задание 4. "Третья страница" заказов
Пагинация списка заказов с размером страницы 2. Запрос показывает третью страницу (заказы 5 и 6) с сортировкой по дате создания, сумме и идентификатору.

### Задание 5. Заказы со статусом paid: первая страница
Комбинированный запрос, включающий фильтрацию по статусу заказа, сортировку по дате и сумме, а также пагинацию для отображения первой страницы результатов.

### Задание 6. Пагинация клиентов
Простая пагинация списка клиентов с размером страницы 2 и сортировкой по идентификатору. Демонстрирует базовый подход к организации постраничного вывода.

## 🎯 Цели проекта

- Освоить синтаксис операторов `ORDER BY`, `LIMIT` и `OFFSET` в PostgreSQL
- Научиться комбинировать сортировку, фильтрацию и пагинацию в одном запросе
- Понять принципы обеспечения стабильности сортировки с помощью уникальных полей
- Освоить формулу расчета смещения для любой страницы результатов
- Подготовиться к решению реальных задач по организации постраничного вывода в приложениях

## 🛠️ Используемые технологии

- **СУБД:** PostgreSQL 12 и выше
- **Язык запросов:** SQL (DDL, DML, DQL)
- **Инструменты:** pgAdmin, DBeaver, psql или любой другой SQL-клиент
