
# SQL Задачи по работе с базами данных

## 📋 Описание проекта

Данный проект содержит коллекцию SQL-запросов для анализа данных интернет-магазина. Запросы охватывают различные аспекты работы с реляционными базами данных: от базовых соединений до сложных агрегаций и проверок целостности данных.

## 🎯 Цели проекта

- Отработка навыков написания SQL-запросов различной сложности
- Изучение различных типов JOIN-операций
- Практика работы с агрегатными функциями и группировкой данных
- Проверка целостности данных между связанными таблицами

## 🗄️ Структура базы данных

Проект использует схему данных интернет-магазина, состоящую из следующих таблиц:

| Таблица | Описание |
|---------|----------|
| **clients** | информация о клиентах (client_id, full_name, city) |
| **orders** | заказы (order_id, created_at, status, total_amount, client_id) |
| **order_items** | позиции заказов (order_item_id, order_id, product_id, qty, price) |
| **products** | товары (product_id, product_name) |

## 📊 Список задач

### 1. Заказы и клиенты (INNER JOIN)
Вывод списка заказов с данными клиентов. Демонстрирует базовое соединение двух таблиц.

### 2. Все клиенты и их заказы (LEFT JOIN)
Вывод всех клиентов с их заказами, включая клиентов без заказов. Показывает использование LEFT JOIN.

### 3. Клиенты без заказов (LEFT JOIN + IS NULL)
Поиск клиентов, которые еще не сделали ни одного заказа. Использует фильтрацию по NULL-значениям.

### 4. Детализация позиций заказов (JOIN 4 таблицы)
Сложный запрос, объединяющий все четыре таблицы для получения полной информации о каждой позиции заказа.

### 5. Сумма позиций vs total_amount (проверка)
Верификация данных: сравнение суммы позиций заказа с общей суммой заказа для выявления расхождений.

### 6. Топ товаров по количеству продаж (GROUP BY + JOIN)
Анализ продаж: определение самых популярных товаров по количеству проданных единиц.

### 7. Выручка по товарам (GROUP BY + JOIN)
Финансовый анализ: расчет выручки по каждому товару на основе проданных позиций.

### 8. Клиенты и количество заказов (LEFT JOIN + GROUP BY)
Подсчет количества заказов для каждого клиента, включая клиентов без заказов.

### 9. Paid-заказы: клиенты и сумма оплаченного (LEFT JOIN с фильтром в ON)
Анализ оплаченных заказов с использованием фильтрации в условии JOIN (не в WHERE).

### 10. Заказы без позиций
Поиск потенциальных проблем с данными: заказы, в которых отсутствуют позиции.

## 🔧 Используемые технологии

- **SQL** — язык структурированных запросов
- **Реляционные базы данных** — работа со связанными таблицами
- **СУБД** — PostgreSQL / MySQL / SQLite (запросы универсальны)

## 📈 Ключевые концепции SQL

| Концепция | Описание |
|-----------|----------|
| **INNER JOIN** | Соединение только совпадающих записей |
| **LEFT JOIN** | Все записи из левой таблицы, совпадающие из правой |
| **GROUP BY** | Группировка данных для агрегации |
| **Агрегатные функции** | SUM, COUNT, AVG для вычислений |
| **COALESCE** | Обработка NULL-значений |
| **Подзапросы** | Вложенные SELECT для фильтрации |

## 🎓 Чему можно научиться

- Правильно выбирать тип JOIN в зависимости от бизнес-требований
- Обрабатывать NULL-значения в агрегациях
- Проверять целостность данных между связанными таблицами
- Оптимизировать запросы с группировкой и сортировкой
- Использовать фильтрацию в условиях JOIN vs WHERE
