## 🗄️ Структура базы данных

### Таблицы

| Таблица | Описание | Основные поля |
|---------|----------|---------------|
| **clients** | Информация о клиентах | id, full_name, city, email, phone, registered_at |
| **products** | Каталог товаров | id, title, category, price, stock |
| **orders** | Заказы | id, client_id, order_date, status, total_amount |
| **order_items** | Позиции заказов | id, order_id, product_id, quantity, price |

### Связи между таблицами
- `orders.client_id` → `clients.id` (один клиент может иметь много заказов)
- `order_items.order_id` → `orders.id` (один заказ может содержать много позиций)
- `order_items.product_id` → `products.id` (один товар может встречаться в многих заказах)

### Статусы заказов
- **new** — новый заказ
- **paid** — оплаченный заказ
- **canceled** — отменённый заказ
- **shipped** — отправленный заказ

## 🔧 Используемые технологии и концепции

### Базовые операции SQL
- **SELECT** — выборка данных из таблиц
- **WHERE** — фильтрация строк
- **ORDER BY** — сортировка результатов
- **LIMIT / OFFSET** — ограничение количества записей и пагинация
- **LIKE** — поиск по шаблону с использованием wildcard символов

### Объединение таблиц
- **INNER JOIN** — только совпадающие записи из обеих таблиц
- **LEFT JOIN** — все записи из левой таблицы, совпадающие из правой
- **Условия соединения (ON)** — ключи связывания таблиц

### Агрегация и группировка
- **COUNT()** — подсчёт количества записей
- **SUM()** — суммирование значений
- **AVG()** — вычисление среднего значения
- **MAX() / MIN()** — нахождение максимума и минимума
- **GROUP BY** — группировка данных
- **HAVING** — фильтрация сгруппированных данных

### Условная агрегация
- **FILTER** — условное выполнение агрегатных функций
- **COALESCE** — замена NULL значений

### Подзапросы
- **EXISTS / NOT EXISTS** — проверка существования записей
- **Скалярные подзапросы** — возврат одного значения
- **Коррелированные подзапросы** — ссылающиеся на внешний запрос

### Обобщённые табличные выражения
- **WITH (CTE)** — создание временных наборов данных
- **Многоступенчатые CTE** — последовательное преобразование данных

### Представления
- **CREATE VIEW** — создание сохранённого представления
- **Использование представлений** — упрощение сложных запросов
