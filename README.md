# HR Candidate Tracking System (Система учёта кандидатов)

[![PostgreSQL](https://img.shields.io/badge/PostgreSQL-16-blue.svg)](https://www.postgresql.org/)


## 📋 Описание проекта

Данный проект представляет собой учебную реализацию реляционной базы данных для HR-отдела компании. Система разработана в рамках практической работы по курсу баз данных и демонстрирует полный цикл разработки реляционной БД: от анализа предметной области до создания работающего SQL-скрипта с тестовыми данными и проверочными запросами.

### Предметная область
Система предназначена для автоматизации процесса найма персонала и позволяет:
- Хранить информацию об открытых вакансиях (должность, описание, отдел, статус)
- Веди учёт кандидатов (контактные данные, резюме)
- Отслеживать отклики кандидатов на вакансии
- Управлять этапами собеседований (воронка найма)
- Фиксировать даты собеседований и обратную связь по кандидатам

## 🏗️ Архитектура базы данных

### Схема данных
![ER Diagram](er_diagram.png)

### Структура таблиц

| Таблица | Описание | Ключевые поля |
|---------|----------|---------------|
| `Vacancy` | Вакансии компании | id, title, department, status |
| `Candidate` | Соискатели | id, full_name, phone, email, resume_link |
| `Interview_Stage` | Справочник этапов отбора | id, stage_name, order_num |
| `Application` | Отклики кандидатов (связующая таблица) | id, vacancy_id, candidate_id, stage_id, applied_date, feedback |

### Связи между таблицами
- **Vacancy** → **Application**: один ко многим (1:N)
- **Candidate** → **Application**: один ко многим (1:N)
- **Interview_Stage** → **Application**: один ко многим (1:N)
- **Candidate** ↔ **Vacancy**: многие ко многим (M:N) через таблицу `Application`

### Нормализация
Модель данных соответствует **третьей нормальной форме (3НФ)**:
- ✅ Все атрибуты атомарны (1НФ)
- ✅ Нет частичных зависимостей от составного ключа (2НФ)
- ✅ Нет транзитивных зависимостей неключевых атрибутов (3НФ)
